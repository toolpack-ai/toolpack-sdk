## Bug fixes

### SlackChannel DM self-suppression

In channel posts, Slack sets `event.user` on bot messages so the existing `botUserId` check suppresses them correctly. In DM threads, Slack omits `event.user` on `bot_message` subtypes and sets `event.bot_id` instead — the `botUserId` comparison could never fire, so the agent processed its own DM responses as new incoming messages.

**Fix:** `auth.test` now also captures `bot_id` (the `B...` identifier) alongside `user_id`. A new `self.botId` field is stored at startup. In `shouldProcessEvent`, an additional check drops any event where `event.bot_id` matches `this.botId`.

Both ids are discovered automatically via `auth.test` — no configuration change required.
