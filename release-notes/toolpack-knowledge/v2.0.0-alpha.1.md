# @toolpack-sdk/knowledge v2.0.0-alpha.1

Date: 2026-05-02

## Highlights
- Hybrid retrieval: combine semantic and keyword search with tunable weighting.
- New data sources: JSON, SQLite, and Postgres.
- New embedder: OpenRouterEmbedder (routes to supported embedding models via OpenRouter).
- Smarter persistent storage path resolution with seamless migration from old path.
- Streaming batch ingestion with progress callbacks and single-item `add()` API.

## Breaking/behavioral changes
- `knowledge_search` default similarity `threshold` is now 0.3 (was 0.7) to improve recall for short queries. You can override per-call.
- `knowledge_search` now uses hybrid retrieval by default (semantic + keyword) for more robust matching; you can set `searchType` explicitly if you prefer a single mode.
- The `PersistentKnowledgeProvider` default base path is now `~/.toolpack/db/knowledge`.
  - If an existing DB is found in the old path `~/.toolpack/knowledge`, it will be used automatically (no manual migration needed).
- Tool runtime logs were removed from `knowledge_search` to reduce console noise. Use your own logging or wrap the tool for diagnostics.

## New features
- Retrieval
  - `searchType`: `semantic` | `keyword` | `hybrid` with `semanticWeight` composition.
  - Hybrid result combiner with thresholding and metadata/vector inclusion options.
- Sources
  - `JSONSource`, `SQLiteSource`, `PostgresSource`.
- Embedders
  - `OpenRouterEmbedder` (alongside existing OpenAI and Ollama embedders).
- Ingestion & operations
  - `streamingBatchSize` option for `Knowledge.create` and `onEmbeddingProgress` callback.
  - `add(content, metadata?)` for on-the-fly additions without full re-sync.
  - Provider `getAllChunks()`/`keywordQuery()` hooks and in-memory fallbacks.

## Improvements
- Safer provider initialization (dimension validation), robust batch fallback to per-chunk embedding on errors.
- Consistent return mapping for tool results: `content`, `score`, `metadata`.

## Migration notes
- If you relied on the higher default threshold, set `threshold: 0.7` explicitly in tool calls.
- For storage paths customized previously, no action is needed; the provider continues to respect an explicit `storagePath`.

## Compatibility
- Node.js >= 20

## Configuration
- Embeddings: use `OpenAIEmbedder` (OpenAI) or `OpenRouterEmbedder` (requires `OPENROUTER_API_KEY`). Anthropic does not provide embeddings.

## Install
- npm i @toolpack-sdk/knowledge@alpha
