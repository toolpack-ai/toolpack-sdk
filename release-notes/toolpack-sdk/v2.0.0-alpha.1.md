# Toolpack SDK v2.0.0-alpha.1

Date: 2026-05-02

## Highlights
- Context window management primitives: token counter, message pruner, rolling summary utilities, and state manager with tests.
- First-class SQLite-backed conversation store (`SQLiteConversationStore`) with optional FTS, plus improved in-memory store.
- New OpenRouter provider adapter and tests.
- Large suite of GitHub tools added (PR review, diffs, files, GraphQL, comments, etc.).
- Refined tool orchestration and provider logging.
- New examples demonstrating context window usage.

## Breaking changes
- Internal conversation modules were restructured. If you imported deep paths directly, update to use the public entry points:
  - Import types and stores from `toolpack-sdk` (re-exported) instead of `toolpack-sdk/src/conversation/...`.
  - Public exports now include:
    - `InMemoryConversationStore`, `SQLiteConversationStore`
    - Types: `ConversationScope`, `StoredMessage`, `GetOptions`, `ConversationStore`, `PromptMessage`, `AssembledPrompt`
- Message pruner behavior: when an assistant message with `tool_calls` is pruned, its paired tool result messages are also removed to avoid orphaned tool results that some providers reject. This is a behavioral change that should be API-compatible and reduce provider errors.

## New features
- Conversation stores
  - `SQLiteConversationStore` (WAL by default, optional FTS, pruning helpers, tested).
  - Reworked `InMemoryConversationStore` built on a generic LRU cache.
- Context window management
  - `utils/token-counter`, `utils/context-window-state`, `utils/message-pruner`, `utils/message-summarizer`, plus related errors and tests.
- Providers
  - New `openrouter` provider adapter and unit tests.
  - Improved provider logger and configuration surface.
- Tools
  - GitHub tool family: content retrieval, PR diffs/files, GraphQL execution, PR reviews, replies, threads, comments, and more.
- Examples
  - `examples/context-window-*.ts` showing basic, monitoring, multi-conversation, and summarization workflows.

## Improvements
- Cleaner public exports for conversation types/stores (`src/conversation/index.ts`).
- More robust history querying/limiting in stores.
- Better handling of duplicate or empty user messages in agent flows (paired work in agents package).

## Configuration and environment
- OpenRouter provider requires `OPENROUTER_API_KEY`.
- GitHub tools require authentication via one of:
  - `GITHUB_PAT` (Personal Access Token), or
  - GitHub App credentials: `GITHUB_APP_ID` and `GITHUB_APP_PRIVATE_KEY` (with installation resolved from `repo`).

## Fixes
- Pruning now removes orphaned tool result messages with their originating assistant turn, preventing provider validation errors.

## Migration notes
- Replace any deep imports from `src/conversation/{store,conv-types,participant}` with the top-level `toolpack-sdk` exports. Example:
  - Before: `import { InMemoryConversationStore } from 'toolpack-sdk/src/conversation/store'`
  - After: `import { InMemoryConversationStore, SQLiteConversationStore } from 'toolpack-sdk'`
- If you relied on keeping tool result messages when pruning assistant turns, note the new coupled removal.

## Compatibility
- Node.js >= 20

## Install
- npm i toolpack-sdk@alpha
