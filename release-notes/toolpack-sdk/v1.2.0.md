# Toolpack SDK v1.2.0 Release Notes

**Release Date:** April 5, 2026  
**Previous Version:** v1.1.0

## Overview
This release introduces a major architectural change with the extraction of the Knowledge module into a separate package, ESM-first dual-build support, and enhanced platform compatibility.

## Breaking Changes

### Knowledge Module Extraction
- **Knowledge functionality moved to `@toolpack-sdk/knowledge`**
- The `knowledge` directory has been removed from `toolpack-sdk`
- SDK now uses duck-typing via `KnowledgeInstance` interface for optional integration
- Import changes required:
  ```typescript
  // Before (v1.1.0)
  import { Knowledge, MemoryProvider } from 'toolpack-sdk/knowledge';
  
  // After (v1.2.0)
  import { Knowledge, MemoryProvider } from '@toolpack-sdk/knowledge';
  ```

## New Features

### ESM-First with Dual Build
- **ESM** (`dist/index.js`) - Primary modern format
- **CJS** (`dist/index.cjs`) - Legacy compatibility
- **TypeScript declarations** (`.d.ts` and `.d.cts`) - Full type support
- Uses `tsup` for unified build process

### MCP (Model Context Protocol) Support
- New MCP client integration for external tool servers
- Configurable via `mcp` option in `Toolpack.init()`
- Support for stdio and HTTP transports

### Enhanced Windows Support
- Full Windows CI testing across all matrix combinations
- PowerShell-compatible test execution
- Cross-platform path handling fixes

## Improvements

### Build & Distribution
- **Minification enabled** - 52% smaller package size (186KB → 38KB)
- **Source maps disabled** for production builds
- **Tree-shaking support** via `sideEffects` field in package.json
- Removed redundant `minify.js` script (esbuild handles this)

### Type System
- `KnowledgeInstance` interface for duck-typed knowledge integration
- Removed unnecessary peer dependency constraints
- Stricter TypeScript configuration

### Code Organization
- Monorepo architecture with workspace packages
- Knowledge module fully extracted to standalone package
- Cleaner dependency graph between SDK and Knowledge

## Bug Fixes

### CI/CD
- Fixed Windows CI test failures
- Resolved PowerShell execution issues
- Cross-platform path normalization for glob patterns

### Dependencies
- Fixed Netlify target version issue (`^24.0.1`)
- Removed peer dependency version conflicts
- Better-sqlite3 native addon compatibility

## Migration Guide

### For Knowledge Users
1. Install the new package:
   ```bash
   npm install @toolpack-sdk/knowledge
   ```

2. Update imports:
   ```typescript
   import { Knowledge } from '@toolpack-sdk/knowledge';
   ```

3. SDK integration remains unchanged:
   ```typescript
   const toolpack = await Toolpack.init({
     provider: 'openai',
     knowledge: kb,  // Still works with duck-typing
   });
   ```

### For SDK-only Users
No changes required. SDK continues to work standalone.
