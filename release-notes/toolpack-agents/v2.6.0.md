# @toolpack-sdk/agents v2.6.0

Date: 2026-07-20

## Highlights

- **TelegramChannel: inline keyboard support** — send messages with interactive buttons and handle button taps as first-class `AgentInput`
- **BaseAgent: labeled `delegate_to_agent` return** — sub-agent results are now wrapped with a `[Response from <agent> — task complete]` label so orchestrators can clearly distinguish a finished delegation from a live conversation message

---

## Breaking changes

None.

---

## New features

### TelegramChannel: inline keyboards and callback queries

**Sending buttons:** `send()` now accepts `metadata.replyMarkup` (a Telegram `InlineKeyboardMarkup`-shaped object). Existing callers that omit `replyMarkup` are unaffected.

```typescript
await channel.send({
  output: 'Approve this trade?',
  metadata: {
    chatId: 123456789,
    replyMarkup: {
      inline_keyboard: [[
        { text: '✅ Approve', callback_data: 'approve:abc-123' },
        { text: '❌ Reject',  callback_data: 'reject:abc-123'  },
      ]],
    },
  },
});
```

**Receiving button taps:** `normalize()` now detects `callback_query` updates and routes them as a distinct path rather than falling through to the `message`/`edited_message` handler. The resulting `AgentInput` carries:

| Context field | Description |
|---|---|
| `isCallback` | Always `true` on callback inputs |
| `callbackQueryId` | Pass to `answerCallbackQuery()` to dismiss the spinner |
| `callbackData` | The `callback_data` string from the tapped button |
| `chatId` | Numeric chat ID |
| `userId` | Numeric user ID of who tapped |
| `username` / `firstName` / `lastName` | Sender identity fields |
| `messageId` | ID of the message that contained the keyboard |

**Dismissing the loading spinner:** new public method `answerCallbackQuery(callbackQueryId, text?)` calls the Telegram `answerCallbackQuery` endpoint. This is not called automatically — the agent is responsible for calling it after handling the tap.

```typescript
async invokeAgent(input: AgentInput): Promise<AgentResult> {
  if (input.context?.isCallback) {
    const result = await this.handleTap(input.context.callbackData as string);
    await (this.channels[0] as TelegramChannel).answerCallbackQuery(
      input.context.callbackQueryId as string,
      'Recorded.',
    );
    return result;
  }
  return this.run(input.message ?? '');
}
```

Throws `Error('Telegram API error: <description>')` if the API rejects the request (e.g. query expired).

## Improvements

### BaseAgent: `delegate_to_agent` return value is now labeled

When using `delegation.mode: 'await'`, the tool result returned to the LLM is now prefixed with `[Response from <agent> — task complete]`. This makes it unambiguous to the orchestrating LLM that it is reading a completed sub-agent result, not a live conversation message, reducing the chance of the orchestrator replying to the sub-agent instead of synthesising the result for the end user.

**Before:** raw `AgentResult` JSON (`{"output":"...","metadata":{...}}`)  
**After:** `{"output":"[Response from browser — task complete]\n\n<original output>","metadata":{...}}`

No code changes required — the label is injected automatically by `BaseAgent` when the tool result is returned.
