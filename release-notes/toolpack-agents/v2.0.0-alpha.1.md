# @toolpack-sdk/agents v2.0.0-alpha.1

Date: 2026-05-02

## Introduction
Initial alpha release of the Agent layer for Toolpack. Build, compose, and deploy AI agents with a consistent, extensible pattern on top of `toolpack-sdk`.

## Highlights
- Core framework
  - `BaseAgent` with event lifecycle, tool orchestration, mode support, and pluggable interceptors.
  - `AgentRegistry` for multi-agent composition and transport.
  - Capability agents: `IntentClassifierAgent`, `SummarizerAgent`.
- Channels (ingress/egress)
  - Slack, Telegram, Discord, Email, SMS, Webhook, and Scheduled (cron) channels.
  - Typed configs and comprehensive tests across channels.
- Conversation history
  - Pluggable history capture via interceptors, addressed-only mode, and conversation-aware prompt assembly.
  - Optional `ConversationSearch` tool exposure per conversation to fetch past turns.
- Interceptor system
  - Dedup, noise filter, self-filter, rate-limit, participant resolver, capture, address-check, depth guard, tracer.
  - Composable chains with strong typing and skip semantics.
- Testing utilities
  - Mocks and helpers to unit-test agents and channels.

## Notable behaviors
- Avoids emitting duplicate inbound user messages in assembled prompts and skips empty user messages (prevents provider errors).
- Designed to pair with `toolpack-sdk` context-window primitives (token counter, pruner, summarizer) for robust long-running conversations.

## Getting started
- Install: `npm i @toolpack-sdk/agents@alpha`
- Minimal code:
```ts
import { AgentRegistry, BaseAgent } from '@toolpack-sdk/agents';
```
- See package README for Slack setup and multi-agent orchestration examples.

## Configuration and environment
- Channels require provider tokens/secrets as applicable (see README examples):
  - Slack: `SLACK_BOT_TOKEN`, `SLACK_SIGNING_SECRET`
  - Telegram: Bot token
  - Discord: Bot token
  - Email: SMTP creds (Nodemailer)
  - SMS: Twilio creds
- If using knowledge tools with agents, ensure embeddings are configured via `OPENAI_API_KEY` or `OPENROUTER_API_KEY` in your app.
- Tool usage can be guided or constrained via the SDK’s mode config (`allowedToolCategories`, `allowedTools`, and tool search settings).

## Compatibility
- Node.js >= 20
- Peer dependencies: `toolpack-sdk@^2.0.0-alpha`, `@toolpack-sdk/knowledge@^2.0.0-alpha` (optional), plus channel-specific SDKs.
