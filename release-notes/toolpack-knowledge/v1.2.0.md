# @toolpack-sdk/knowledge v1.2.0 Release Notes

**Release Date:** April 5, 2026  
**Initial Package Release**

## Overview
This is the **initial release** of `@toolpack-sdk/knowledge` as a standalone package, extracted from the main `toolpack-sdk` monolith. This modularization allows for independent versioning, focused development, and optional adoption.

## What's New

### Package Structure
- **New standalone package** `@toolpack-sdk/knowledge`
- **ESM + CJS dual build** via tsup
- **Full TypeScript support** with declaration files
- **Zero external SDK dependency** - works independently

### Core Components

#### Knowledge Providers
- `MemoryProvider` - In-memory vector storage for development
- `PersistentKnowledgeProvider` - SQLite-backed storage with WAL mode

#### Embedders
- `OpenAIEmbedder` - OpenAI text-embedding models with retry logic
- `OllamaEmbedder` - Local embeddings via Ollama (nomic-embed-text, mxbai-embed-large, etc.)

#### Sources
- `MarkdownSource` - Heading-based chunking with frontmatter extraction
  - Code block detection
  - Deterministic chunk IDs
  - Configurable chunk sizes and overlap

### Key Features

#### RAG Pipeline
```typescript
const kb = await Knowledge.create({
  provider: new MemoryProvider(),
  sources: [new MarkdownSource('./docs/**/*.md')],
  embedder: new OllamaEmbedder({ model: 'nomic-embed-text' }),
  description: 'Documentation search',
});
```

#### Tool Integration
```typescript
// Returns a tool compatible with Toolpack SDK
const tool = kb.toTool();
// Tool has: name, description, parameters, execute()
```

#### Error Handling
- `KnowledgeError` - Base error class
- `EmbeddingError` - Embedding API failures
- `IngestionError` - Source parsing failures
- `DimensionMismatchError` - Vector dimension mismatches
- `KnowledgeProviderError` - Provider operation failures

#### Metadata Filtering
```typescript
await kb.query('search', {
  filter: { 
    category: 'api',
    hasCode: { $in: [true] },
    priority: { $gt: 5 }
  }
});
```

## API Reference

### Knowledge.create()
```typescript
interface KnowledgeOptions {
  provider: KnowledgeProvider;
  sources: KnowledgeSource[];
  embedder: Embedder;
  description: string;
  reSync?: boolean;
  onError?: ErrorHandler;
  onSync?: SyncEventHandler;
  onEmbeddingProgress?: EmbeddingProgressHandler;
}
```

### query()
```typescript
await kb.query(text: string, options?: {
  limit?: number;        // default: 10
  threshold?: number;    // default: 0.7
  filter?: MetadataFilter;
  includeMetadata?: boolean;
  includeVectors?: boolean;
});
```

- `better-sqlite3` - SQLite database

## Browser/Node Compatibility
- **Node.js >= 20** required
- Native SQLite (better-sqlite3) - Node.js only
- No browser support (by design for server/CLI use)

## Breaking Changes from SDK v1.1.0
If migrating from the old embedded knowledge module:

```typescript
// Old (SDK v1.1.0)
import { Knowledge } from 'toolpack-sdk/knowledge';

// New (standalone package)
import { Knowledge } from '@toolpack-sdk/knowledge';
```

All other APIs remain compatible.

## Integration with Toolpack SDK
```typescript
import { Toolpack } from 'toolpack-sdk';
import { Knowledge } from '@toolpack-sdk/knowledge';

const kb = await Knowledge.create({...});

const toolpack = await Toolpack.init({
  provider: 'anthropic',
  knowledge: kb,  // Duck-typed integration
});
```
