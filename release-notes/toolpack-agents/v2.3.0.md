# @toolpack-sdk/agents v2.3.0

Date: 2026-06-13

## Highlights

- **Full `ToolpackInitConfig` on `BaseAgent`** — agents can now pass `interceptors`, `knowledge`, `customModes`, `customTools`, and all other init options directly in their constructor config, not just `apiKey`/`provider`/`model`
- **Per-request mode forwarded to `generate()`** — `BaseAgent.run()` passes the agent's mode per-request, enabling the SDK's mode race fix to protect concurrent agents sharing a Toolpack instance

---

## Breaking changes

None. `BaseAgentOptions` previously accepted `{ apiKey, provider?, model? }` — that shape is still valid as a subset of `ToolpackInitConfig`.

---

## New features

### Full `ToolpackInitConfig` accepted by `BaseAgent`

`BaseAgentOptions` now accepts the full `ToolpackInitConfig` union instead of the narrow `{ apiKey, provider?, model? }` shape. This means agents that create their own Toolpack instance (rather than sharing one from `AgentRegistry`) can now configure it fully at construction time:

```typescript
class MyAgent extends BaseAgent {
  constructor() {
    super({
      apiKey: process.env.ANTHROPIC_KEY!,
      provider: 'anthropic',
      model: 'claude-sonnet-4-6',
      interceptors: [createSkillInterceptor({ dir: './skills', minScore: 1.0 })],
      knowledge: [myKnowledgeBase],
      customModes: [publisherMode],
    });
  }
}
```

Previously `interceptors`, `knowledge`, and `customModes` could only be passed to agents that received a pre-configured shared `toolpack` instance. They can now be set directly on any agent.

---

## Bug fixes

### Mode not forwarded per-request in `BaseAgent.run()`

`BaseAgent.run()` called `this.toolpack.generate()` without setting `mode` on the request. The SDK's new per-request mode snapshot (see `toolpack-sdk` v2.3.0) requires the mode to be present on the request to be immune to concurrent `setMode()` calls. `run()` now passes `mode: this.mode` on every `generate()` call, so the agent's mode is snapshotted at the start of each request and cannot be overridden by a concurrently executing sub-agent.

## Compatibility

- Node.js >= 20
- Peer dependency `toolpack-sdk` `^2.3.0`
- Peer dependency `@toolpack-sdk/knowledge` `^2.3.0` (optional)
