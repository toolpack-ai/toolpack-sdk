# @toolpack-sdk/agents v2.0.0

Date: 2026-05-24

## Highlights

- **Agent Mind** — a persistent cognitive layer for `BaseAgent`. Agents can now store goals, beliefs, and reflections across runs using a SQLite-backed (or custom) knowledge store. Seven LLM-callable mind tools are automatically injected when enabled.
- **Scheduler** — a full SQLite-backed job scheduler (`SchedulerStore`) with cron and one-shot scheduling, plus four LLM-callable `createSchedulerTools()` for agents to manage jobs at runtime.
- **Discord channel** — new `DiscordChannel` implementation for two-way Discord bot integration.
- **Interceptors in BaseAgent** — agents now support the same interceptor middleware pattern as the SDK, enabling per-agent request/response transformation.
- **`BaseChannel` type alias removed** — the deprecated `BaseChannel = ChannelInterface` alias is gone; use `ChannelInterface` directly.

---

## Breaking changes

- **`BaseChannel` type alias removed** from `agent/types.ts`. Replace `BaseChannel` with `ChannelInterface` in all type annotations. The real `BaseChannel` abstract class (from `channels/base-channel.ts`) is unchanged and still exported.
- **`better-sqlite3` peer dep aligned to `^12.6.2`** (was `^11.x`). Upgrade your `better-sqlite3` installation if on v11.
- **`toolpack-sdk` peer dep updated to `^2.0.0`** (was `^2.0.0-alpha.1`).
- **`@toolpack-sdk/knowledge` peer dep updated to `^2.0.0`** (was `^2.0.0-alpha.1`).
- **`publishConfig.tag` changed from `alpha` to `latest`**.

## New features

### Agent Mind

Persistent cognitive memory for `BaseAgent`. When enabled, the mind:
1. Prepends an `--- AGENT MIND ---` header to the system prompt on every `run()` call (active goals, pinned rules, recent reflections, high-confidence beliefs).
2. Injects 7 LLM-callable mind tools into every request.
3. Flushes the draft buffer to the store when the run completes.

```typescript
class MyAgent extends BaseAgent {
  mind = {
    provider: new PersistentKnowledgeProvider({ namespace: 'mind/my-agent' }),
    maxGoals: 5,
    maxPinnedReflections: 5,
    retrievalThreshold: 0.35,
    ttlDefaults: { beliefTtlDays: 30, reflectionTtlDays: 90 },
  };
}
```

**Mind tools (auto-injected when `mind` is configured):**

| Tool | Purpose |
|---|---|
| `mind_recall` | Semantic search across goals, beliefs, and reflections |
| `mind_believe` | Store a new belief with optional confidence and TTL |
| `mind_reflect` | Store a reflection (optionally pin as standing rule) |
| `mind_unpin_reflection` | Unpin a standing-rule reflection |
| `mind_set_goal` | Create a new goal with priority and tags |
| `mind_update_goal` | Update an existing goal's description or progress |
| `mind_complete_goal` | Mark a goal as completed (immediate commit, bypasses draft buffer) |

All write operations (except `mind_complete_goal`) are buffered per-run and committed on completion — writes from within a run are not visible to `mind_recall` until the next run.

### Scheduler

SQLite-backed job scheduling with full cron and one-shot support:

```typescript
import { SchedulerStore, createSchedulerTools } from '@toolpack-sdk/agents';

const scheduler = new SchedulerStore({ dbPath: '.toolpack/scheduler.db' });

// LLM-callable tools: scheduler_list, scheduler_create, scheduler_update, scheduler_delete
const tools = createSchedulerTools(scheduler);
```

- Supports cron expressions and `runAt` (ISO timestamp) for one-shot jobs.
- Persistent SQLite store with WAL mode.
- `ScheduledChannel` integration for triggering agents on schedule.
- `cron-expressions` helper for human-readable cron descriptions.

### Discord channel

New `DiscordChannel` for two-way Discord bot integration:

```typescript
import { DiscordChannel } from '@toolpack-sdk/agents';

const discord = new DiscordChannel({
  token: process.env.DISCORD_TOKEN,
  guildId: 'your-guild-id',
});
```

- Handles slash commands and direct messages.
- Thread-aware: sends replies to the originating thread when available.
- Startup self-check validates bot token on connect.
- Graceful fallback from thread send to channel send.

### Interceptors in BaseAgent

Agents now accept an `interceptors` array in their options, using the same `ToolpackInterceptor` interface from `toolpack-sdk`:

```typescript
class MyAgent extends BaseAgent {
  interceptors = [myLoggingInterceptor, mySkillInterceptor];
}
```

## Improvements

- **`BaseAgent`** — significant refactoring: cleaner delegation flow, mind integration, interceptor wiring, improved `onBeforeRun`/`onComplete`/`onError` lifecycle hooks.
- **`ScheduledChannel`** — major rewrite with improved reliability, proper timezone handling, next-run calculation, and detailed run logging.

- **`SlackChannel`** — improved startup self-check and request handling.
- **`AgentTypes`** — new types exported: `AgentMindConfig`, `MindGoal`, `MindBelief`, `MindReflection`, `MindRecallResult`, `SchedulerJob`, `SchedulerToolsConfig`.

## Migration notes

**`BaseChannel` type alias:**
```typescript
// Before
import type { BaseChannel } from '@toolpack-sdk/agents';
const myChannel: BaseChannel = ...;

// After
import type { ChannelInterface } from '@toolpack-sdk/agents';
const myChannel: ChannelInterface = ...;
```

**`better-sqlite3` upgrade:**
```bash
npm install better-sqlite3@^12.6.2 @types/better-sqlite3
```
