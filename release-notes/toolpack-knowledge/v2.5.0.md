## New features

### `rateLimitMs` option on `VertexAIEmbedder`

Large knowledge base syncs that call `embedBatch` in a tight loop can exhaust the Vertex AI embedding QPM quota, causing `RESOURCE_EXHAUSTED` errors even with retry logic. The new `rateLimitMs` option enforces a minimum elapsed time between consecutive `embedBatch` calls:

```typescript
import { VertexAIEmbedder } from '@toolpack-sdk/knowledge';

const embedder = new VertexAIEmbedder({
  model: 'gemini-embedding-001',
  rateLimitMs: 500, // 0.5 s between batches ‚Üí ~120 QPM effective rate
});
```

When set, `embedBatch` checks how long ago the previous call completed. If less than `rateLimitMs` has elapsed, it waits for the remainder before proceeding. The timer starts after the request completes (not before it starts), so the delay is between calls, not between start times.

Default is `0` (no delay ‚Äî existing behaviour unchanged).

The existing `retryDelay` option description has also been updated to note that `RESOURCE_EXHAUSTED` / quota errors benefit from a much longer retry delay (60,000 ms+) compared to transient network errors.
