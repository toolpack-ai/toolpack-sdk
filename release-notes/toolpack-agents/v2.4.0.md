# @toolpack-sdk/agents v2.4.0

Date: 2026-06-28

## Highlights

- **Agent Spawning** — LLM-driven ephemeral sub-agents: configure templates on `BaseAgent.spawn` and the SDK injects `spawn_agent` / `spawn_agents_parallel` tools into every `run()` call; the LLM picks a template, hands off a task, gets the result back, and continues — all within a single conversation turn
- **Hot Reload & Graceful Restart** — `HotReloadWatcher` watches `.ts`/`.tsx` and `.env*` files, debounces rapid edits (default 30 s), runs `tsc --build` on TypeScript changes, then calls `registry.scheduleRestart()` which waits for all in-flight conversations to finish before calling `process.exit(0)`
- **Dynamic agent management** — `AgentRegistry.addAgent()` and `removeAgent()` allow agents to be wired and started (or stopped and unregistered) at runtime after the registry is already running
- **`EphemeralAgent` exported** — the lightweight one-shot agent used internally by the spawn tools is now a public export for direct use in tests and advanced orchestration

## New features

### Agent Spawning

Set `spawn` on any `BaseAgent` to let the LLM instantiate one-off helper agents on demand:

```typescript
import { BaseAgent } from '@toolpack-sdk/agents';
import type { AgentSpawnConfig } from '@toolpack-sdk/agents';

class OrchestratorAgent extends BaseAgent {
  name = 'orchestrator';
  description = 'Coordinates research and coding tasks';
  mode = 'agent';

  spawn: AgentSpawnConfig = {
    enabled: true,
    templates: [
      {
        name: 'researcher',
        description: 'Searches the web and summarises findings on a topic.',
        systemPrompt: (task) => `You are a focused research agent. Task: ${task}`,
      },
      {
        name: 'coder',
        description: 'Writes or refactors code for a specific request.',
        systemPrompt: (task) => `You are a senior engineer. Task: ${task}`,
        model: 'claude-opus-4-8',
      },
    ],
    maxDepth: 3,
  };

  async invokeAgent(input) {
    return this.run(input.message);
  }
}
```

When `run()` executes, two tools are injected into the LLM's tool list:

- **`spawn_agent`** — spawn one helper agent and wait for its result
- **`spawn_agents_parallel`** — spawn multiple agents simultaneously via `Promise.all`

Spawned agents are **ephemeral**: no channels, no registry entry, discarded after their task completes. They inherit the parent's `spawn` config so they can themselves spawn (up to `maxDepth`).

**Template options:**

| Option | Type | Description |
|--------|------|-------------|
| `name` | `string` | Unique identifier. Use `'self'` for self-replication. |
| `description` | `string` | Purpose shown to the LLM so it can pick the right template. |
| `systemPrompt` | `(task: string) => string` | Called at spawn time with the task string. |
| `model` | `string` | Model override. Inherits parent when omitted. |
| `allowPromptAddition` | `boolean` | Allow the LLM to append extra instructions via `systemPromptAddition`. Default: `false`. |

**Self-replication:** add `{ name: 'self', ... }` to templates. The replica inherits the parent's mode and system prompt with the template's `systemPrompt(task)` result appended.

**Depth control:** at `maxDepth`, the spawn tools are silently omitted — no error is thrown. Default is `3`.

**`systemPromptAddition` in tool schema:** the `systemPromptAddition` parameter only appears in the `spawn_agent` and `spawn_agents_parallel` tool schemas when at least one template has `allowPromptAddition: true`. Templates without the flag silently ignore any value the LLM passes.

### Hot Reload & Graceful Restart

`HotReloadWatcher` watches source directories for changes and triggers a graceful restart:

```typescript
import { AgentRegistry, HotReloadWatcher } from '@toolpack-sdk/agents';

const registry = new AgentRegistry([myAgent]);
await registry.start();

const watcher = new HotReloadWatcher({
  watchPaths: ['./src'],
  cwd: process.cwd(),
  debounceMs: 30_000,           // default: 30 s of silence before acting
  onRestartNeeded: () => registry.scheduleRestart(),
  onCompileError: (msg) => console.error('[tsc]', msg),
});
watcher.start();
```

**How it works:**

1. A file change is detected in a watched path.
2. The debounce timer resets. Every new change resets it again.
3. After the debounce window of silence:
   - `.ts` / `.tsx` → runs `npx tsc --build`. On exit 0, calls `onRestartNeeded`. On non-zero, calls `onCompileError` (no restart triggered).
   - `.env*` → calls `onRestartNeeded` directly (no compile step).
4. `onRestartNeeded` calls `registry.scheduleRestart()`.
5. The registry checks `isAllIdle()` immediately. If all agents are idle, restarts right away. Otherwise waits for `agent:complete` events.
6. After a deadline (`maxWaitMinutes`, default 30), forces the restart even if conversations are still active.
7. `_executeRestart()` stops all agents, drains conversation locks (30 s deadline, 100 ms poll), then calls `process.exit(0)`.
8. PM2 / systemd brings the process back up with the new `dist/` and `.env`.

`watcher.stop()` closes all file watchers and cancels any pending debounce timer.

**`scheduleRestart` options:**

```typescript
registry.scheduleRestart({
  maxWaitMinutes: 30, // default — force restart after this many minutes even if conversations are still active
});
```

`scheduleRestart()` is idempotent — subsequent calls are no-ops.

### Dynamic agent management

`AgentRegistry` now supports adding and removing agents after `start()`:

```typescript
// Add — wired + started immediately if the registry is already running
const newAgent = new ResearchAgent({ apiKey: process.env.ANTHROPIC_API_KEY! });
await registry.addAgent(newAgent);

// Remove — stops the agent and unregisters its channels
await registry.removeAgent('old-agent');

// Check if all agents are currently idle
if (registry.isAllIdle()) {
  console.log('Safe to restart');
}
```

`addAgent()` before `start()` defers wiring until the registry starts (existing behaviour).

### `EphemeralAgent` — now public

`EphemeralAgent` is a concrete `BaseAgent` with no channels, no interceptors, and no registry entry. It is used internally by the spawn tools and is now exported for direct use in tests or advanced orchestration:

```typescript
import { EphemeralAgent } from '@toolpack-sdk/agents';

// Standalone (pass apiKey; call start() before invokeAgent)
const agent = new EphemeralAgent(
  'helper',
  'One-off summariser',
  { ...AGENT_MODE, systemPrompt: 'Return bullet points only.' },
  { apiKey: process.env.ANTHROPIC_API_KEY! },
);
await agent.start();
const result = await agent.invokeAgent({ message: 'Summarise: ...' });

// Inside a BaseAgent subclass (share the parent's toolpack)
class MyAgent extends BaseAgent {
  async invokeAgent(input) {
    const e = new EphemeralAgent('helper', 'desc', mode, { toolpack: this.toolpack });
    return e.invokeAgent({ message: input.message });
  }
}
```
