# Toolpack SDK v2.0.0

Date: 2026-05-24

## Highlights

- **Skills system** — define reusable behavioral instructions in `.skill.md` files and auto-inject them into agent requests via BM25 relevance search using the new `createSkillInterceptor`.
- **Interceptors** — a new `ToolpackInterceptor` middleware layer lets you transform requests and responses in `generate()` calls. Skills are the first built-in use of interceptors.
- **Skill tools** — four LLM-callable tools (`skill.create`, `skill.read`, `skill.update`, `skill.list`) let agents manage the skill library at runtime.
- **VertexAI provider** — first-class Google VertexAI provider adapter alongside the existing Gemini adapter.
- **`git.clone` tool** — new git tool for cloning repositories.
- **20 missing barrel exports fixed** — `fsGlob`, `fsBatchRead`, `fsBatchWrite`, `fsDeleteDir`, five web tools, nine coding tools, `slackConversationsReplies`, and `githubIssuesCommentsCreate` are now importable directly from `toolpack-sdk`.
- **Deprecated APIs removed** — `intelligentToolDetection` config and `log()` function fully removed.

---

## Breaking changes

- **`intelligentToolDetection` removed** from `ToolsConfig`. If you had `toolsConfig.intelligentToolDetection` set, remove it. Use `toolSearch` for on-demand tool discovery instead.
- **`IntelligentToolDetectionConfig` interface removed** from public API. Remove any type imports of this interface.
- **`log()` function removed** from `provider-logger`. Replace any calls with `logInfo()`, `logDebug()`, etc.
- **`publishConfig.tag` changed from `alpha` to `latest`**. If you pin the tag in install scripts, update to `npm install toolpack-sdk`.

## New features

### Skills module

A new `skills/` module enables retrieval-augmented instruction injection:

```typescript
import { Toolpack, createSkillInterceptor, createSkillTools } from 'toolpack-sdk';

const toolpack = await Toolpack.init({
  provider: 'anthropic',
  interceptors: [
    createSkillInterceptor({ dir: '.toolpack/skills', maxSkills: 3, minScore: 0.3 }),
  ],
  tools: [
    createSkillTools({ dir: '.toolpack/skills' }),
  ],
});
```

- Skills are `.skill.md` files with a structured frontmatter (`name`, `title`, `description`, `tags`, `triggers`) and an `## Instructions` section.
- The interceptor runs a BM25 search on every user message and prepends matched skill instructions as a `<skill-instructions>` block.
- `createSkillInterceptor` validates all skill files at `Toolpack.init()` time (configurable `onValidationError: 'fail' | 'warn'`).
- BM25 field weighting: name/title ×3, tags/triggers ×2, description ×1.
- Mtime-based cache invalidation — the index auto-rebuilds only when files change.
- Concurrent-build safety via internal promise lock.

### Interceptors

New `ToolpackInterceptor` callable type for middleware in `generate()`:

```typescript
import type { ToolpackInterceptor, ToolpackNextFunction } from 'toolpack-sdk';

const myInterceptor: ToolpackInterceptor = async (request, next) => {
  // transform request, call next, transform response
  return next(request);
};

// Optional startup hook — called during Toolpack.init()
myInterceptor.init = async () => { /* eager validation */ };
```

Pass interceptors via `Toolpack.init({ interceptors: [...] })`.

### VertexAI provider

New `vertexai` provider adapter for Google Vertex AI. Configure with `GOOGLE_CLOUD_PROJECT` and `GOOGLE_CLOUD_LOCATION` env vars or pass credentials directly.

### `git.clone` tool

New `git.clone` tool for cloning remote repositories with configurable depth and branch targeting.

### Barrel export fixes

The following tools were defined but not re-exported from the `toolpack-sdk` root — now fixed:

| Category | Newly exported |
|---|---|
| `fs-tools` | `fsGlobTool`, `fsDeleteDirTool`, `fsBatchReadTool`, `fsBatchWriteTool` |

## Improvements

- **`client/index.ts`** — major internal cleanup: removed the AI-based tool inference subsystem (`isAfterToolCall`, `inferNeedsToolsWithAI`, `extractToolContext`, `getFastModelForProvider`) that backed the now-removed `intelligentToolDetection` feature. Tool routing is now purely regex-based, faster and lighter.
- **`Toolpack.init()`** — now wires up interceptors and calls `interceptor.init()` for each interceptor with a startup hook.
- **`tool-orchestrator.ts`** — improvements to parallel tool execution.
- **`tools/index.ts`** — 21 new exports (20 fixes + `skill-tools`).
- **`git` tools** — improved `blame`, `diff`, and `log` schemas.

---

## Fixes

- Tool barrel exports missing for 20 tools across 5 categories (see above).
- `git.blame`, `git.diff`, and `git.log` schema refinements.

## Removals

- `IntelligentToolDetectionConfig` interface

- `log()` function from `provider-logger` (use `logInfo()` etc.)
- `isAfterToolCall`, `inferNeedsToolsWithAI`, `extractToolContext`, `getFastModelForProvider` internal functions in client

## Migration notes

**If you used `intelligentToolDetection`:**
```typescript
// Before
const toolpack = await Toolpack.init({
  toolsConfig: {
    intelligentToolDetection: { enabled: true, maxFollowUpMessages: 5 },
  },
});

// After — use toolSearch for on-demand tool discovery
const toolpack = await Toolpack.init({
  toolsConfig: {
    toolSearch: { enabled: true, searchResultLimit: 5 },
  },
});
```

**If you imported `log()` from provider-logger:**
```typescript
// Before
import { log } from 'toolpack-sdk/providers/provider-logger';
log('message');

// After
import { logInfo } from 'toolpack-sdk/providers/provider-logger';
logInfo('message');
```
